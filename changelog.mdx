# Release Candidates

# Changelog

## 4.3.0 - 2024-03-13

### Eliminado

- El parametro `acquirerId` en la request `GET /claims/acquirer/{acquirerId}` fue eliminado. Ahora el ID se obtiene usando la api key.
  Por lo que se tendría que cambiar de `GET /claims/acquirer/{acquirerId}` a `GET /claims/acquirer`.

## 4.2.21 - 2024-03-06

### Cambiado

- Solo se aceptan formatos `JPG`, `TIFF`, `JPEG` para la subida de documentos.

## 4.2.15 - 2024-03-04

### Añadido

- Campo `createdAt` para representar el momento de creación del caso de chargeback.
- Arreglo `events` para rastrear eventos significativos en el proceso de chargeback, incluyendo sus nombres y marcas de tiempo.
- Campo `id` como identificador único del caso de chargeback.
- Campo `partnerId` para identificar al socio involucrado en la transacción.
- Campo `seen` para indicar si el caso de chargeback ha sido visto.
- Campo `status` para representar el estado actual del caso de chargeback.
- Campo `updatedAt` para representar el último momento de actualización del caso de chargeback.
- Campo `currency` para representar la moneda de la transacción.
- Campo `amount` para representar el monto de la transacción.
- Campo `rrn` para representar el número de referencia de recuperación.
- Campo `acquirerId` para representar el identificador del adquirente, en el root de la disputa.
- Campo `issuerId` para representar el identificador del emisor, en el root de la disputa.
- Información detallada de `address` dentro del objeto `merchant`, incluyendo `notes`, `number`, `street`, `city`, `country`, `id`, `name`, `state`, y `zip_code`.
- El objeto `merchant` ahora incluye `mcc`, `name`, `paymentData` con `card_holder_doc_number`, `expiration_month`, `name`, `pan`, `product_id`, y actualiza `taxId` y `taxIdType`.

### Cambiado

- `amount` cambiado a tipo numérico.
- `isOpen` cambiado de string a booleano.
- `primaryAccountNumber` actualizado a formato enmascarado.

### Eliminado

- Campos `openClaims`, `lastModifiedDate`, `refundReversalCurrency`, `lastModifiedBy`, `claimId`, `chargebackReferenceNumber`, `merchantId`, `issuerResponseCode`, `issuerRejectReasonCode`, `outcome`, `authCode` dentro del objeto `merchant`.

## 4.1.9 - 2024-02-26

### Fixed

- Se corrigió un error en el endpoint de `POST /documentation` que devolvía una url incorrecta, que corrompía los archivos subidos.
- Se mejora el procesamiento por IA de documentos.
