# Checklist para ir a producción

Este es un checklist de cosas que hay que hacer para ir a producción con nuestro servicio.
Es probable que algunos pasos no apliquen a tu proyecto, pero es importante que los revises.

## Infraestructura

Los items para revisar en la infraestructura dependen de si estás usando un proveedor de cloud o no.

- [ ] Revisar que la infraestructura esté en un ambiente de producción
- [ ] Tener una infraestructura con alta disponibilidad (multi región, multi zona, etc.)
- [ ] Tener un plan de disaster recovery

## Código

Los siguientes items son deseables.

- [ ] Tener un plan de rollback
- [ ] Revisar que no haya información sensible en el código (contraseñas, llaves, etc.)
- [ ] Revisar que no haya información sensible en el historial de commits
- [ ] Manejar los errores retornados por nuestra API de una forma que te permita actuar acorde
- [ ] Usar un secret manager. Obtener las credenciales para usar el servicio con servicios como AWS Secrets Manager o Vault

## Monitoreo

Los siguientes items son deseables y permiten que puedas actuar de forma proactiva.

- [ ] Tener un dashboard de monitoreo con las métricas más importantes
- [ ] Alarmado de errores por SMS, Slack, OpsGenie, etc.

## Partnership

Items que hay que revisar con Kausanna.

- [ ] Tener un acuerdo de partnership con Kausanna
- [ ] Tener a mano el contacto de responsable de tu cuenta Kausanna y el de soporte
